# Варианты реализации по модулям

> Для каждого модуля — несколько подходов с плюсами и минусами.

---

## Модуль «Рецептура»

### Вариант A: Простой справочник
- Рецепт = список ингредиентов с количествами
- При изменении — просто перезаписываем
- **+** Просто
- **-** Теряется история, старые партии будут показывать новую себестоимость

### Вариант B: Версионирование (рекомендуется)
- Каждое изменение рецепта — новая версия
- Партия ссылается на конкретную версию
- **+** Точная историческая себестоимость
- **-** Чуть сложнее в реализации

---

## Модуль «Склад» — метод списания

### Вариант A: FIFO (First In, First Out)
- Списываем сначала то, что пришло раньше
- **+** Точная себестоимость, стандарт бухучёта
- **-** Сложнее реализовать

### Вариант B: Средневзвешенная цена
- При каждом приходе пересчитываем среднюю цену
- Формула: (старый_остаток × старая_цена + приход × цена_прихода) / (старый_остаток + приход)
- **+** Простая реализация, нет «партий» в сырье
- **-** Менее точно

### Вариант C: По последней цене
- Себестоимость = цена последней закупки
- **+** Максимально просто
- **-** Неточно, не рекомендуется для бухучёта

---

## Модуль «Продажи» — ценообразование

### Вариант A: Фиксированная цена на продукт
- Одна цена для всех клиентов
- **+** Просто
- **-** Нет гибкости

### Вариант B: Прайс-листы
- Несколько прайс-листов (опт, розница, VIP)
- Клиент привязан к прайс-листу
- **+** Гибко, удобно
- **-** Нужно поддерживать несколько прайсов

### Вариант C: Индивидуальные цены
- Для каждого клиента — свои цены
- **+** Максимальная гибкость
- **-** Сложно управлять при большом количестве клиентов

---

## Модуль «Зарплата» — расчёт

### Вариант A: Фиксированный оклад
- Зарплата / кол-во рабочих дней × отработанные дни + премии - штрафы
- **+** Просто, покрывает 80% случаев

### Вариант B: Оклад + сдельная часть
- Фиксированная часть + оплата за произведённые партии/литры
- **+** Мотивирует сотрудников
- **-** Сложнее расчёт, нужна привязка к производству

### Вариант C: Полностью сдельная
- Только оплата за результат
- **+** Чёткая связь работа → оплата
- **-** Сложно, нестабильный доход для работников

---

## Уведомления — каналы

### Вариант A: Только в системе
- Колокольчик с числом непрочитанных уведомлений
- **+** Просто, не нужны внешние сервисы
- **-** Увидят только когда откроют систему

### Вариант B: Система + Telegram-бот
- Критичные уведомления дублируются в Telegram
- **+** Оперативно, все пользуются Telegram
- **-** Нужен бот, настройка

### Вариант C: Система + Email
- **+** Стандартный подход
- **-** Email часто не читают оперативно

---

## Отчёты — формат

### Вариант A: Только на экране
- Таблицы и графики в интерфейсе
- **+** Быстро реализовать
- **-** Нельзя распечатать или отправить

### Вариант B: Экран + экспорт в Excel
- Все табличные отчёты можно скачать как .xlsx
- **+** Привычный формат, можно дальше анализировать
- **-** Нужна библиотека генерации Excel

### Вариант C: Экран + Excel + PDF
- Для официальных документов (накладные, расчётки) — PDF
- **+** Полноценно
- **-** Больше работы на генерацию шаблонов
