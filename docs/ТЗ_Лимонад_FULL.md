## 1. Общая информация

### 1.1 Цель проекта

Разработка веб-системы внутреннего учёта для сети лимонадного производства с несколькими цехами (филиалами) в разных городах. Система охватывает полный цикл: от закупки сырья и производства до продажи готовой продукции, включая финансовый учёт (расходы, прибыль, зарплата). Данные ведутся по каждому филиалу раздельно с возможностью консолидированного просмотра.

### 1.2 Платформа

Веб-приложение, доступное через браузер. Серверная часть разворачивается на одном центральном сервере (облако), доступном из всех городов. Адаптивная вёрстка для доступа с планшетов.

### 1.3 Пользователи системы

| Роль | Описание |
|------|----------|
| Директор | Полный доступ ко всем модулям. Видит финансовые показатели, управляет сотрудниками. |
| Бухгалтер | Расходы, зарплата, отчёты. Не управляет производством. |
| Менеджер по продажам | Продажи, клиенты, склад готовой продукции. Не видит зарплаты. |
| Кладовщик | Приход/расход сырья, остатки. Не видит финансы. |

### 1.4 Масштаб

Текущий: 2+ цеха (филиала) в разных городах, до 10 видов продукции, до 10 сотрудников на филиал, до 50 клиентов на филиал.
Система должна масштабироваться: новые филиалы, рост ассортимента, сотни клиентов.

## 2. Глоссарий

| Термин | Определение |
|--------|------------|
| **Рецептура** | Описание состава одного вида лимонада: список ингредиентов с количествами на определённый объём партии. |
| **Версия рецептуры** | Снимок состава рецепта на момент изменения. Старые партии ссылаются на старую версию. |
| **Производственная партия** | Единичный акт производства: выбранная рецептура, списанное сырьё, полученная продукция. |
| **Ингредиент** | Любое сырьё, входящее в состав продукции: сахар, вода, ароматизатор, лимонная кислота и т.д. |
| **Тара** | Упаковочные материалы: бутылки, крышки, этикетки. Учитываются как отдельная категория сырья. |
| **Себестоимость** | Сумма затрат на сырьё + тару для производства единицы/партии продукции. Рассчитывается в сомони по курсу на дату производства. |
| **Средневзвешенная цена** | Метод расчёта стоимости списания: (остаток × старая_цена + приход × новая_цена) / (остаток + приход). Для сырья ведётся в USD. |
| **Soft delete** | Пометка записи как удалённой без физического удаления из БД. Используется для финансовых данных. |
| **Дебиторская задолженность** | Сумма, которую клиенты должны за отгруженный, но не оплаченный товар. |
| **Валюта закупки (USD)** | Доллар США — валюта, в которой закупается сырьё и тара. |
| **Валюта продажи (TJS)** | Сомони — национальная валюта Таджикистана, в которой ведутся продажи, расходы и зарплата. |
| **Курс обмена** | Соотношение USD/TJS на конкретную дату. Используется для пересчёта себестоимости сырья в сомони. |
| **Курсовая разница** | Разница в стоимости сырья на складе, возникающая при изменении курса USD/TJS. |
| **Филиал** | Обособленное производственное подразделение (цех) в конкретном городе. Имеет свой склад, сотрудников, производство и продажи. |
| **Межфилиальный трансфер** | Перемещение сырья или готовой продукции между складами разных филиалов. |
| **Консолидированный отчёт** | Отчёт, объединяющий данные всех филиалов. |

## 3. Модуль «Филиалы и структура»

### 3.1 Концепция

Система поддерживает **2 и более филиалов** (цехов) в разных городах. Каждый филиал — это автономная производственная единица со своим складом, сотрудниками и продажами. При этом управление централизовано: единая база данных, единые справочники (рецептуры, ингредиенты, статьи расходов), единый курс валют.

### 3.2 Сущность: Филиал

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Название | строка | да | Например: «Цех Душанбе», «Цех Худжанд» |
| Код | строка | да | Короткий код для нумерации документов: `DSH`, `HDZ` |
| Город | строка | да | |
| Адрес | строка | нет | |
| Телефон | строка | нет | |
| Ответственный | ссылка | нет | Руководитель филиала |
| Статус | enum | да | Активный / Закрыт |

### 3.3 Что общее, что раздельное

| Сущность | Уровень | Пояснение |
|----------|---------|-----------|
| Справочник ингредиентов | **Общий** | Одинаковый сахар во всех филиалах |
| Справочник рецептур | **Общий** | Рецепты едины для всей сети |
| Прайс-листы | **По филиалу** | Цены могут отличаться по городам |
| Курсы валют | **Общие** | Единый курс на дату |
| Склад сырья | **По филиалу** | У каждого свои остатки и средневзвешенные цены |
| Склад готовой продукции | **По филиалу** | У каждого свои остатки |
| Производственные партии | **По филиалу** | Производство привязано к конкретному цеху |
| Клиенты | **Общие** | Клиент может покупать у любого филиала |
| Продажи | **По филиалу** | Продажа происходит из конкретного филиала |
| Расходы | **По филиалу** + **Общие** | Аренда — по филиалу, зарплата директора — общий |
| Сотрудники | **По филиалу** | Каждый сотрудник закреплён за филиалом (кроме директора) |
| Пользователи / Роли | **По филиалу** или **Все** | Директор — все филиалы, менеджер — свой |

### 3.4 Межфилиальные трансферы

#### Сущность: Трансфер

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Номер | строка | авто | `ТР-КОД_ОТКУДА-YYYYMMDD-NNN` |
| Дата создания | дата | да | |
| Филиал-отправитель | ссылка | да | |
| Филиал-получатель | ссылка | да | |
| Тип | enum | да | Сырьё / Готовая продукция |
| Статус | enum | да | Создан → В пути → Получен → Отменён |
| Дата отправки | дата | авто | При смене статуса на «В пути» |
| Дата получения | дата | авто | При смене статуса на «Получен» |
| Отправил | ссылка | авто | |
| Принял | ссылка | нет | Заполняется при приёмке |
| Комментарий | текст | нет | |

#### Позиции трансфера

| Поле | Тип | Описание |
|------|-----|----------|
| Ингредиент / Продукт | ссылка | В зависимости от типа трансфера |
| Количество отправлено | число | |
| Количество получено | число | Может отличаться (потери при транспортировке) |
| Расхождение | число | авто |

#### Бизнес-правила трансферов

- При отправке: списание со склада филиала-отправителя.
- При получении: приход на склад филиала-получателя.
- Если количество получено < отправлено — разница фиксируется как потери при транспортировке.
- **Себестоимость при трансфере:** товар передаётся по средневзвешенной цене (USD) филиала-отправителя. Средневзвешенная цена на складе получателя пересчитывается.
- Пока трансфер «В пути» — товар не числится ни на одном складе (учитывается отдельно как «В пути»).
- Отменить можно только со статуса «Создан». Если уже «В пути» — только принять (даже с нулевым количеством и комментарием).

### 3.5 Переключение контекста филиала

- В шапке интерфейса — **переключатель филиала** (dropdown).
- Директор и бухгалтер видят опцию **«Все филиалы»** (консолидированный вид).
- Менеджер и кладовщик видят только свой филиал (или филиалы, к которым привязаны).
- Все операции (создание продаж, приход сырья, производство) выполняются **в контексте выбранного филиала**.
- Переключение филиала меняет все списки, остатки, отчёты.

## 4. Роли и права доступа

### 4.1 Аутентификация

- Вход по логину и паролю.
- Пароль: минимум 8 символов.
- Блокировка учётной записи после 5 неудачных попыток входа (разблокировка — директором).
- Сессия: автоматический выход после 8 часов неактивности.

### 4.2 Привязка пользователя к филиалу

| Поле | Тип | Описание |
|------|-----|----------|
| Пользователь | ссылка | |
| Роль | enum | Директор / Бухгалтер / Менеджер / Кладовщик |
| Филиалы | список ссылок | К каким филиалам имеет доступ |
| Область видимости | enum | **Все филиалы** / **Только свои** |

- **Директор**: область видимости = «Все филиалы». Может переключаться между любыми + видеть консолидацию.
- **Бухгалтер**: область видимости = «Все филиалы». Видит финансы по всем филиалам.
- **Менеджер по продажам**: привязан к 1+ филиалу. Видит и создаёт продажи только в своих филиалах.
- **Кладовщик**: привязан к 1 филиалу. Работает только со складом своего филиала.

### 4.4 Журнал действий (аудит)

Каждое создание, изменение и удаление записи логируется:
- Кто (пользователь)
- Когда (дата и время)
- Что (сущность + ID)
- Действие (создание / изменение / удаление)
- Филиал (в контексте которого выполнено действие)
- Старое и новое значение (для изменений)

Журнал доступен директору. Фильтр по филиалу. Записи журнала не удаляются.

## 5. Модуль «Валюты и курсы»

### 5.1 Принцип мультивалютности

Система работает с двумя валютами:

| Контекст | Валюта | Обозначение |
|----------|--------|-------------|
| Закупка сырья и тары | Доллар США | USD ($) |
| Продажи | Сомони | TJS (смн.) |
| Операционные расходы (аренда, коммуналка, транспорт) | Сомони | TJS (смн.) |
| Зарплата | Сомони | TJS (смн.) |
| Себестоимость продукции | Хранится в USD, **отображается в обеих** | USD + TJS |
| Отчёты (прибыль, P&L) | Сомони (основная) + USD (справочно) | TJS (смн.) |

### 5.2 Справочник курсов

#### Сущность: Курс валюты

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Дата | дата | да | Одна запись на дату |
| Валютная пара | enum | да | USD/TJS (расширяемо) |
| Курс | число (decimal 18,4) | да | Сколько TJS за 1 USD |
| Источник | enum | да | Ручной / Нацбанк РТ |
| Кто установил | ссылка | авто | Пользователь |

#### Бизнес-правила

- **Обязательный курс на каждый рабочий день.** Если курс не установлен — система берёт последний известный курс и выводит предупреждение: «Курс USD/TJS на [дата] не установлен, используется курс от [дата]: X TJS».
- Курс устанавливается **вручную** директором или бухгалтером (поле в настройках или экран «Курсы»).
- На будущее (v2): автоматическая загрузка курса с сайта Национального банка Таджикистана.
- Историю курсов нельзя удалить — только добавить новую запись.
- Изменение курса **не** пересчитывает ранее проведённые документы. Каждый документ фиксирует курс на свою дату.

### 5.3 Где используется курс

| Операция | Валюта ввода | Как пересчитывается |
|----------|-------------|---------------------|
| Приход сырья | USD | Сумма в TJS = Сумма в USD × курс на дату прихода |
| Себестоимость партии | USD (из средневзвешенной цены сырья) | Себестоимость в TJS = Себестоимость в USD × курс на дату производства |
| Продажа | TJS | Для отчёта по марже: выручка TJS vs себестоимость (USD → TJS по курсу на дату продажи) |
| Расход (ручной) | TJS | Хранится как есть |
| Зарплата | TJS | Хранится как есть |
| Отчёт P&L | TJS | Все суммы приводятся к TJS по курсу на дату каждой операции |

### 5.4 Отображение в интерфейсе

- Все суммы, связанные с сырьём: показывать **обе валюты** — `$120.00 (1 308.00 смн.)`.
- Все суммы, связанные с продажами, расходами, зарплатой: только **сомони**.
- В отчётах: колонки `Сумма (USD)` и `Сумма (TJS)` для сырья; только `Сумма (TJS)` для остального.
- На дашборде: прибыль и выручка в **сомони**.
- Текущий курс отображается в шапке системы: `1 USD = XX.XX TJS (дата)`.

## 6. Модуль «Рецептура и производство»

> **Рецептуры** — общие для всех филиалов (единый справочник).
> **Производственные партии** — привязаны к конкретному филиалу.

### 6.1 Справочник рецептур

#### Сущность: Рецептура

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Название | строка | да | Название вида лимонада |
| Описание | текст | нет | Комментарий, заметки |
| Объём партии | число | да | Объём в литрах, на который рассчитан состав |
| Выход бутылок | число | да | Количество бутылок с одной партии |
| Объём бутылки | число | да | Литров в одной бутылке (0.33 / 0.5 / 1.0) |
| Статус | enum | да | Активна / Архивная |
| Текущая версия | ссылка | авто | Ссылка на актуальную версию состава |

#### Сущность: Версия рецептуры

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Рецептура | ссылка | да | К какому продукту относится |
| Номер версии | число | авто | Автоинкремент при каждом изменении состава |
| Дата создания | дата | авто | Когда создана эта версия |
| Автор | ссылка | авто | Кто изменил |
| Комментарий | текст | нет | Причина изменения |

#### Сущность: Состав версии (рецепт → ингредиенты)

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Версия рецептуры | ссылка | да | |
| Ингредиент | ссылка | да | Из справочника ингредиентов |
| Количество | число | да | На один объём партии |
| Единица измерения | enum | да | кг / г / л / мл / шт |

#### Сущность: Тара для рецептуры

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Версия рецептуры | ссылка | да | |
| Материал | ссылка | да | Бутылка / крышка / этикетка из справочника |
| Количество на партию | число | да | |

#### Вычисляемые поля

- **Себестоимость партии (USD)** = сумма (количество × средневзвешенная цена в USD) по всем ингредиентам + тара
- **Себестоимость партии (TJS)** = Себестоимость (USD) × курс USD/TJS на дату производства
- **Себестоимость 1 бутылки (USD)** = себестоимость партии (USD) / выход бутылок
- **Себестоимость 1 бутылки (TJS)** = себестоимость партии (TJS) / выход бутылок
- **Себестоимость 1 литра (USD)** = себестоимость партии (USD) / объём партии

#### Бизнес-правила

- При изменении состава рецептуры создаётся новая версия. Старая версия сохраняется и привязана к уже произведённым партиям.
- Архивная рецептура не отображается в списке выбора при создании партии, но остаётся в истории.
- Себестоимость (USD) пересчитывается при изменении средневзвешенных цен на сырьё.
- Себестоимость (TJS) пересчитывается при изменении курса или цен. Для уже произведённых партий курс зафиксирован.

|------|-----|-------------|----------|
| Номер партии | строка | авто | Формат: `П-КОД_ФИЛИАЛА-YYYYMMDD-NNN` (напр. `П-DSH-20260201-001`) |
| Филиал | ссылка | да | Из текущего контекста пользователя |
| Рецептура | ссылка | да | |
| Версия рецептуры | ссылка | авто | Фиксируется текущая версия в момент создания |
| Дата производства | дата | да | |
| Срок годности | дата | авто | Дата производства + срок хранения продукта |
| Запланированный объём | число | да | Количество партий по рецептуре |
| Фактический выход (бутылки) | число | да | Заполняется по факту |
| Брак (бутылки) | число | нет | По умолчанию 0 |
| Технологические потери (л) | число | нет | Нормальные потери при производстве |
| Статус | enum | да | Запланирована → В производстве → Завершена → Отменена |

| Себестоимость партии (USD) | деньги | авто | Сумма ингредиентов + тара в USD |
| Себестоимость партии (TJS) | деньги | авто | USD × курс на дату производства |
| Ответственный | ссылка | нет | Сотрудник, выполнявший производство |
| Комментарий | текст | нет | |

#### Автоматическое списание сырья

При переводе партии в статус **«В производстве»**:

1. Система проверяет достаточность всех ингредиентов и тары **на складе данного филиала**.
2. **Если хватает** — автоматическое списание со склада сырья по составу рецептуры × запланированный объём.
3. **Если НЕ хватает** — система показывает:
   - Какого ингредиента не хватает
   - Сколько есть / сколько нужно
   - Дефицит

   Пользователь может:
   - Отменить создание
   - Уменьшить объём партии
   - Подтвердить принудительно (если ингредиент вот-вот придёт) — в этом случае остаток может уйти в минус, запись помечается как «с дефицитом»

#### При завершении партии

- Фактический выход + брак фиксируются.
- Готовая продукция приходуется на склад готовой продукции **того же филиала**.
- Если фактический выход сильно отличается от запланированного (>10%), система выводит предупреждение.

#### При отмене партии

- Если сырьё уже списано (статус был «В производстве»), оно возвращается на склад.
- Обязательный комментарий: причина отмены.

#### Учёт себестоимости — метод средневзвешенной цены в USD

Средневзвешенная цена ведётся **в долларах** (USD), **отдельно по каждому филиалу**, т.к. закупки в долларах:

```
новая_средняя_USD = (текущий_остаток × текущая_средняя_USD + количество_прихода × цена_прихода_USD)
                    / (текущий_остаток + количество_прихода)
```

Эта цена (в USD) используется для расчёта себестоимости при производстве. Пересчёт в сомони происходит по курсу на дату производственной партии.

**Почему в USD, а не в TJS:** курс доллара меняется, и если пересчитывать каждый приход в сомони на дату прихода, средневзвешенная в TJS будет смешивать разные курсы. Хранение в USD даёт точную картину: сколько стоит сырьё в валюте закупки, и пересчёт в TJS делается один раз — на дату использования.

| Сумма (оригинальная валюта) | деньги | да | В валюте расхода |
| Курс USD/TJS | число | авто | Заполняется для расходов в USD |

> **Примечание:** все расходы для целей отчётности приводятся к TJS. Расходы на сырьё (автоматические из прихода) хранятся в USD и пересчитываются в TJS по курсу на дату прихода.

#### Автоматические расходы

- **Приход сырья**: при оформлении прихода автоматически создаётся расход по статье «Сырьё» / «Тара».
- **Зарплата**: при утверждении расчёта зарплаты — автоматический расход по статье «Зарплата».
- Автоматические расходы нельзя редактировать — только просматривать. Редактируется исходный документ.

#### Повторяющиеся расходы

- Для аренды, коммуналки и подобных: создаётся шаблон повторяющегося расхода.
- Система ежемесячно (в указанный день) автоматически создаёт расход-черновик.
- Пользователь подтверждает или корректирует сумму.
- Неподтверждённые черновики — в уведомлениях.

#### Формулы расчёта

**Оклад:**
```
Базовая = Оклад / Рабочих_дней_в_месяце × Отработано_дней
```

**Почасовая:**
```
Базовая = Ставка_за_час × Отработано_часов
```

**Сдельная:**
```
Базовая = Ставка_за_единицу × Количество_произведённых_единиц
```
(Количество берётся из производственных партий, где сотрудник указан ответственным)

#### Бизнес-правила

- Расчёт создаётся как черновик, может быть отредактирован.
- После утверждения — создаётся автоматический расход по статье «Зарплата».
- Утвердить может только директор.
- Нельзя утвердить расчёт, если не заполнен табель хотя бы за 20 дней месяца (предупреждение).

## 13. Нефункциональные требования

### 13.1 Производительность

- Загрузка любой страницы — не более 2 секунд при нормальной нагрузке.
- Формирование отчёта — не более 5 секунд.
- Экспорт в Excel — не более 10 секунд для данных за год.

### 13.2 Надёжность

- Все финансовые данные используют **soft delete** (не удаляются физически).
- **Автоматический бэкап** базы данных — ежедневно. Хранение — 30 дней.
- Транзакционность: списание со склада и создание расхода — в одной транзакции. Если одно не прошло, откат обоих.

### 13.3 Безопасность

- Пароли хранятся в виде bcrypt-хэша.
- HTTPS обязателен.
- CSRF-защита на все формы.
- SQL-инъекции: использование параметризованных запросов / ORM.
- XSS: экранирование всего пользовательского ввода при отображении.
- Rate limiting на эндпоинт логина.

### 13.4 Совместимость

- Браузеры: Chrome (последние 2 версии), Firefox (последние 2 версии), Edge (последние 2 версии).
- Адаптивная вёрстка: корректное отображение на экранах от 1024px (планшет горизонтально).
- Мобильные телефоны: не приоритет, но базовая навигация должна работать.

### 13.5 Масштабируемость

- Добавление нового филиала — через интерфейс (создание записи + назначение пользователей), без изменения кода.
- Справочники (статьи расходов, категории, прайс-листы) расширяемы пользователем.
- API-first подход: вся логика доступна через REST API, фронтенд общается только через API.
- Все запросы к данным фильтруются по `facility_id` на уровне сервера — данные одного филиала не утекают в другой.

### 13.6 Локализация

- Интерфейс: русский язык.
- **Валюты:**
  - Основная: сомони (TJS, смн.). Используется для продаж, расходов, зарплаты, отчётов.
  - Закупочная: доллар США (USD, $). Используется для сырья и тары.
  - Хранение: decimal(18,4) для курсов, decimal(18,2) для сумм.
- Часовой пояс: Asia/Dushanbe (UTC+5) в БД и отображении.
- Формат даты: ДД.ММ.ГГГГ.
- Разделитель тысяч: пробел. Десятичный: запятая.
- Формат валют: `1 234,56 смн.` / `$1,234.56`.

### Допущения

- 2 и более филиалов в разных городах, у каждого свой склад.
- Все филиалы работают в одном часовом поясе (Asia/Dushanbe, UTC+5).
- Две валюты: USD (закупки сырья) и TJS (продажи, расходы, зарплата).
- Курс USD/TJS единый для всех филиалов, устанавливается вручную ежедневно.
- Изменение курса не пересчитывает ранее проведённые документы — каждый документ хранит курс на свою дату.
- Справочники (рецептуры, ингредиенты, клиенты, курсы) — общие для всех филиалов.
- Остатки склада, производство, продажи, расходы, сотрудники — по филиалу.
- Система используется для внутреннего учёта, а не для формирования бухгалтерской отчётности.
- Количество одновременных пользователей: до 5 на филиал + 5 управления.

[Запланирована] → [В производстве] → [Завершена]
       ↓                  ↓
  [Отменена]         [Отменена]

```

```

### Приложение В: Формулы

**Средневзвешенная цена сырья (USD):**
```
Новая_средняя_USD = (Остаток × Старая_средняя_USD + Приход × Цена_прихода_USD)
                    / (Остаток + Приход)
```

**Себестоимость производственной партии:**
```
Себестоимость_USD = Σ (Количество_ингредиента × Средневзвешенная_цена_USD)
                   + Σ (Количество_тары × Средневзвешенная_цена_тары_USD)

Себестоимость_TJS = Себестоимость_USD × Курс_USD_TJS_на_дату_производства

Себестоимость_1_бутылки_USD = Себестоимость_USD / Выход_бутылок
Себестоимость_1_бутылки_TJS = Себестоимость_TJS / Выход_бутылок
```

**Валовая прибыль по продукту (в TJS):**
```
Валовая прибыль = Цена_продажи_TJS − Себестоимость_единицы_TJS
Маржа % = (Валовая прибыль / Цена_продажи_TJS) × 100
```

**Чистая прибыль за период (в TJS):**
```
Чистая прибыль = Σ Выручка_TJS − Σ Все_расходы_TJS
```
*Где расходы на сырьё (USD) пересчитаны в TJS по курсу на дату каждой операции.*

**Пересчёт расхода на сырьё из USD в TJS:**
```
Расход_на_сырьё_TJS = Сумма_прихода_USD × Курс_USD_TJS_на_дату_прихода
```

### Приложение Г: Форматы нумерации

> Все документы включают **код филиала** для однозначной идентификации.

| Документ | Формат | Пример (Душанбе / Худжанд) |
|----------|--------|--------|
| Производственная партия | `П-КОД-YYYYMMDD-NNN` | П-DSH-20260201-001 / П-HDZ-20260201-001 |
| Продажа | `ПР-КОД-YYYYMMDD-NNN` | ПР-DSH-20260201-015 |
| Возврат | `ВЗ-КОД-YYYYMMDD-NNN` | ВЗ-DSH-20260201-003 |
| Приход сырья | `ПС-КОД-YYYYMMDD-NNN` | ПС-HDZ-20260201-007 |
| Межфилиальный трансфер | `ТР-КОД_ОТКУДА-YYYYMMDD-NNN` | ТР-DSH-20260201-002 |

### Приложение Д: Схема статусов межфилиального трансфера

```
[Создан] → [В пути] → [Получен]
    ↓                      ↓
[Отменён]           (если расхождение >
 (до отправки)       → уведомление директору)
```

**Поток:**
1. Кладовщик филиала A создаёт трансфер → статус «Создан»
2. Кладовщик филиала A подтверждает отправку → «В пути», товар списан со склада A
3. Кладовщик филиала B принимает → «Получен», товар приходуется на склад B
4. Если принятое кол-во < отправленного → разница = потери, уведомление директору

### Приложение Ж: Пример расчёта себестоимости с двумя валютами

**Исходные данные:**
- Рецептура «Классический лимонад», партия 100 л, выход 200 бутылок (0.5 л)
- Курс на дату производства: 1 USD = 10.90 TJS

| Ингредиент | Кол-во | Ед. | Средневзв. цена (USD) | Сумма (USD) |
|------------|--------|-----|----------------------|-------------|
| Сахар | 15 | кг | $0.80 | $12.00 |
| Лимонная кислота | 2 | кг | $3.50 | $7.00 |
| Ароматизатор | 0.5 | л | $12.00 | $6.00 |
| Вода | 85 | л | $0.02 | $1.70 |
| **Итого сырьё** | | | | **$26.70** |
| Бутылка 0.5л | 200 | шт | $0.05 | $10.00 |
| Крышка | 200 | шт | $0.02 | $4.00 |
| Этикетка | 200 | шт | $0.03 | $6.00 |
| **Итого тара** | | | | **$20.00** |
| **ИТОГО** | | | | **$46.70** |

**Результат:**
- Себестоимость партии: **$46.70** = **509.03 смн.** (× 10.90)
- Себестоимость 1 бутылки: **$0.2335** = **2.55 смн.**
- Цена продажи (базовый прайс): **8.00 смн./бут**
- Маржа: **5.45 смн.** = **68.1%**
